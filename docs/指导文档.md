# 指导文档（前后端对接）

## 1. 后端项目分析（MyGitHub）

后端位于：`MyGitHub/`，技术栈：

- FastAPI + SQLAlchemy(Async) + PostgreSQL
- 统一响应结构：`{ code, message, data }`
- 鉴权：`Authorization: Bearer <access_token>`（JWT）
- 重要约束：`/api/v1/users`、`/api/v1/groups/*` 依赖 `get_current_verified_user`，**要求邮箱已验证**，否则返回 `403`

### 1.1 关键接口（与前端已对接）

认证（前缀：`/api/v1/auth`）

- `POST /register`：注册（返回 access/refresh + user）
- `POST /login`：登录（返回 access/refresh + user）
- `POST /logout`：登出（refresh token 吊销）
- `POST /refresh`：刷新 access/refresh
- `GET /email/status`：获取邮箱验证状态（只要求登录）
- `POST /email/verify`：邮箱验证（只需要 token）
- `POST /email/resend`：重发验证邮件（只需要 email）

用户（前缀：`/api/v1/users`，**需邮箱已验证**）

- `GET /api/v1/users`：用户列表
- `POST /api/v1/users`：新建用户（包含 password）
- `PUT /api/v1/users/{id}`：更新用户（不含 password）
- `DELETE /api/v1/users/{id}`：删除用户（软删除）

分组（前缀：`/api/v1/groups/`，**需邮箱已验证**）

- `GET /api/v1/groups/`：分组列表（包含 `parent_id`）
- `POST /api/v1/groups/`：新建分组
- `PUT /api/v1/groups/{id}`：更新分组（支持移动节点 `parent_id`）
- `DELETE /api/v1/groups/{id}`：删除分组

### 1.2 本地运行要点（后端）

后端依赖 `MyGitHub/.env` 的数据库连接与 CORS：

- `DATABASE_URL=postgresql+asyncpg://...`
- `CORS_ORIGINS=["http://localhost:5173"]`（确保包含前端地址）

> 后端 README 已包含接口示例与说明；如需建表，可运行 `MyGitHub/init_db.py`。

## 2. 前端项目结构（MyGitHub_front）

技术栈：

- Vue 3 + TypeScript + Vue Router + Pinia
- Axios（封装在 `src/lib/http.ts`，带 Bearer token；401 自动 refresh 重试一次）
- Tailwind CSS（统一样式；`@tailwindcss/forms`）

目录要点：

- `src/lib/api.ts`：接口封装（auth/user/group）
- `src/stores/auth.ts`：登录态（localStorage 持久化）
- `src/router/index.ts`：路由 + 守卫（后台需要登录；后台接口页需要邮箱验证）
- `src/pages/admin/*`：后台页面（用户/分组树）

## 3. 页面说明

### 3.1 登录/注册

- 登录：`/auth/login`
- 注册：`/auth/register`

注册成功后会跳转到邮箱验证提示页（后端默认要求验证后才能访问用户/分组接口）。

### 3.2 邮箱验证

提供两种方式：

- 邮件链接打开：`/verify-email?token=...`（无需登录）
- 登录后验证：`/auth/verify-email`（用于后台守卫拦截后引导）

### 3.3 后台（用户管理）

入口：`/admin/users`

- 列表展示用户基础字段、启用状态、邮箱验证状态
- 支持：新建 / 编辑 / 删除
- 分组下拉来自 `GET /api/v1/groups/`

### 3.4 后台（分组管理/树）

入口：`/admin/groups`

- 左侧树：由 `GET /api/v1/groups/` 返回的 `parent_id` 构建
- 支持：新建根分组 / 新建子分组 / 编辑 / 删除

## 4. 启动与联调

1. 启动后端（示例）

```bash
cd MyGitHub
pip install -r requirement.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

2. 启动前端

```bash
cd MyGitHub_front
npm install
npm run dev
```

3. 确认 CORS

确保后端 `.env` 中 `CORS_ORIGINS` 包含 `http://localhost:5173`。
如果你是用 `http://127.0.0.1:5173` 访问前端，也需要把 `http://127.0.0.1:5173` 加进去。

## 5. 可配置项

前端环境变量（`.env.local`）：

- `VITE_API_BASE_URL`：后端地址（默认 `http://127.0.0.1:9000`）
